# lore/

This directory contains historical artifacts: design notes, analyses, and findings frozen at the time they were written.
They document how things were understood at a point in time, not how things are now.

Do not maintain, update, or "fix" files in this directory during refactors or other changes. Code references, file
paths, and described behavior in these documents are allowed to go stale — that is the point. Do not treat them as
current documentation; for the current behavior contract, see `SPEC.md` at the repo root.
